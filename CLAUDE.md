# Project Preferences

## Git

**Branches:** never use "claude" in branch names. use short, informative names (e.g. `update-readme`, `add-backtracking`, `fix-heap`). for small changes push to main directly.

**Commits:** short, lowercase, no em dashes. descriptive but brief. match the repo's casual style (e.g. `update readme: 81/150, add links and difficulty`).
